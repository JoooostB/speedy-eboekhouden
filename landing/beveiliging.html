Beveiliging

Jouw gegevens, jouw controle.

Speedy e-Boekhouden is ontworpen met een fundamenteel uitgangspunt: wij hoeven je gegevens niet te hebben om je te helpen. Op deze pagina leggen we precies uit hoe dat werkt — technisch, eerlijk en zonder marketingtaal.

🔑

Het kernprincipe: Zero-Storage

Speedy slaat geen wachtwoorden, geen e-mailadressen, geen MFA-codes en geen boekhoudkundige gegevens op. Niet in een database, niet in bestanden, niet in logs. Er is simpelweg geen database. Je inloggegevens worden doorgestuurd naar e-boekhouden.nl en daarna direct uit het geheugen verwijderd.

Hoe het technisch werkt

Een transparante blik op elke beveiligingslaag.

🔐

Inloggegevens

Wanneer je inlogt, stuurt je browser je e-mailadres en wachtwoord naar onze server. Onze server stuurt deze onmiddellijk door naar e-boekhouden.nl via een versleutelde HTTPS-verbinding.

Jouw browser → Speedy server → e-boekhouden.nl

Na het doorsturen worden je e-mailadres en wachtwoord nergens bewaard. Ze bestaan alleen gedurende het fractie-van-een-seconde moment dat ze worden doorgestuurd. Daarna worden ze door het besturingssysteem uit het geheugen verwijderd.

✓ Wachtwoord wordt nooit opgeslagen
✓ E-mailadres wordt nooit opgeslagen
✓ MFA-code wordt nooit opgeslagen
🍪

Sessies & cookies

Na een succesvolle login maakt e-boekhouden.nl een sessie-token aan (vergelijkbaar met wanneer je zelf inlogt op hun website). Speedy bewaart dit token tijdelijk in het werkgeheugen (RAM) van de server — niet op schijf, niet in een database.

Jouw browser ontvangt een eigen sessie-cookie van Speedy om je te herkennen. Deze cookie is beveiligd met:

✓ HttpOnly — JavaScript op de pagina kan deze cookie niet uitlezen. Dit beschermt tegen XSS-aanvallen.
✓ Secure — De cookie wordt alleen verstuurd over versleutelde HTTPS-verbindingen.
✓ Willekeurige ID — De sessie-ID is een cryptografisch willekeurig UUID, niet te raden of te voorspellen.
⏱️

Automatische sessieverval

Sessies verlopen automatisch na 30 minuten inactiviteit. Dit is een glijdende termijn: elke actie die je uitvoert zet de klok opnieuw. Maar zodra je 30 minuten niets doet, is de sessie voorbij.

Daarnaast draait er elke 5 minuten een achtergrondproces dat verlopen sessies actief opruimt uit het geheugen. Na opruiming is het sessie-token van e-boekhouden permanent verwijderd — het bestaat nergens meer.

Bij uitloggen wordt je sessie onmiddellijk vernietigd en de cookie ongeldig gemaakt. Als de server herstart gaan alle sessies verloren — er is geen persistente opslag.

🔒

Versleuteling

Alle communicatie is versleuteld met TLS (Transport Layer Security):

✓ Browser ↔ Speedy: HTTPS met een Let's Encrypt certificaat. Onversleuteld verkeer (HTTP) wordt automatisch doorgestuurd naar HTTPS.
✓ Speedy ↔ e-boekhouden.nl: Alle API-aanroepen gaan uitsluitend via HTTPS. Er is geen HTTP-fallback in de code.

Dit betekent dat je gegevens op geen enkel moment onversleuteld over het internet gaan.

📱

Twee-factor-authenticatie (MFA)

Speedy ondersteunt volledig de MFA van e-boekhouden.nl. Als je account MFA heeft ingeschakeld, word je na het inloggen gevraagd om je verificatiecode in te voeren.

De MFA-code wordt — net als je wachtwoord — direct doorgestuurd naar e-boekhouden.nl en daarna uit het geheugen verwijderd. Speedy slaat de code niet op en kan deze niet opnieuw gebruiken.

Wij raden sterk aan om MFA in te schakelen op je e-boekhouden.nl account voor optimale beveiliging.

📊

Welke gegevens kan Speedy inzien?

Speedy vraagt alleen de minimale gegevens op die nodig zijn om het urenformulier te vullen:

✓ Leest: Lijst medewerkers, projecten en activiteiten
✓ Schrijft: Ureninvoer die jij expliciet verstuurt
✗ Kan niet bij je financiële administratie
✗ Kan niet bij facturen of relaties
✗ Kan niet bij accountinstellingen

Speedy kan alleen doen wat jouw e-boekhouden account mag. Er worden geen extra rechten aangevraagd.

🛡️

Cross-Origin beveiliging (CORS)

De API van Speedy accepteert alleen verzoeken van het eigen domein (speedy-eboekhouden.nl). Verzoeken van andere websites worden geblokkeerd. Dit voorkomt dat kwaadwillige websites via jouw sessie gegevens kunnen opvragen of uren kunnen invoeren.

🗄️

Geen database

Speedy heeft geen database. Geen MySQL, geen PostgreSQL, geen MongoDB, geen Redis. Er is letterlijk geen plek waar gegevens permanent opgeslagen zouden kunnen worden.

Alle data (medewerkers, projecten, activiteiten) wordt on-the-fly opgehaald bij e-boekhouden.nl en direct naar je browser gestuurd. Na het versturen wordt de data niet gecachet of bewaard.

Als de server herstart, is alle informatie verdwenen. Er is niets om te hacken, te lekken of te stelen.

Tip: gebruik een beperkt account

Vertrouw je het niet volledig? Maak in e-boekhouden.nl een apart gebruikersaccount aan dat uitsluitend uren mag invoeren. Geef dit account geen toegang tot financiële gegevens, facturen of accountinstellingen. Gebruik vervolgens dit beperkte account om in te loggen bij Speedy. Zo kan Speedy — zelfs in een worst-case scenario — alleen uren invoeren en helemaal niets anders.
